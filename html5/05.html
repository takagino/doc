4. アウトライン・アルゴリズム

前回、要素をグループ化するための要素「div」を勉強したが、「マークアップの効率向上のためのdiv」「デザイン上どうしてもグループ化しないといけないdiv」「全体の構造をわかりやすくするためのdiv」など、様々な用途の「div」が混在すると、HTML全体の見通しが悪くなる。

そこで、HTMLバージョン5から、構造を意味する要素が多数追加された。

4-1. レイアウトを区切るための要素

Webページは基本的に以下の図のようなレイアウトで構成されている。

【header（ヘッダー）】
  • サイトのメインタイトルやロゴ
  • グルーバルナビゲーション（各ページへのリンク）
  • 主要ベージへのリンク
  • ページ内検索フォーム　など、全ページにわたって表示したい内容
【main（メインコンテンツ）】
  • ページの内容
【footer（フッター）】
  • サイトマップ
  • 住所などの基本情報
  • 著作権に関する情報　など、全ページにわたって表示したい内容

HTMLバージョン5より、レイアウトの構成を指定するため、その名前通りの要素が追加された。

<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>EDUCATION</title>
  <link rel="stylesheet" href="./css/style.css">
</head>
<body>
  <!-- ヘッダー -->
  <header>
    <p><img src="images/ico_menu.png"></p>
    <h1><img src="images/img_logo.png"></h1>
  </header>

  <!-- メインコンテンツ -->
  <main>
    <h2>WORLD NEWS</h2>
    <p>Amazing places in America to visit.</p>
    <p>For some reason — this country, this city, this neighborhood, this particular street —  is the place you are living a majority of your life in.</p>
    <a href="#">LEARN MORE →</a>

    <h2>MORE NEWS</h2>
    <p>TRAVEL</p>
    <h3>Article title</h3>
    <p>Lorem ipsum dolor sit amet, ipsum labitur lucilius mel id, ad has appareat…</p>
    <p>2m ago</p>

    <p>TECHNOLOGY</p>
    <h3>Article title</h3>
    <p>Lorem ipsum dolor sit amet, ipsum labitur lucilius mel id, ad has appareat…</p>
    <p>1h ago</p>

    <h2>TRENDING</h2>
    <img src="images/pic_trending01.png">
    <h3>Dolore magna aliqua</h3>
    <p>Lorem ipsum dolor sit amet, ipsum labitur lucilius mel id, ad has appareat.</p>
    <p>2m ago</p>

    <img src="images/pic_trending02.png">
    <h3>Morbi eleifend a libero</h3>
    <p>Lorem ipsum dolor sit amet, ipsum labitur lucilius mel id, ad has appareat.</p>
    <p>1h ago</p>
  </main>

  <!-- フッター -->
  <footer>
    <p>2019 All Rights Reserved.</p>
  </footer>
</body>
</html>

4-2. 文書構造化のための要素

HTMLバージョン5より、文書の構造を指定するための要素が追加された。これらをまとめて「セクショニング・コンテンツ」と呼ぶ。

【section（セクション）要素】
HTML文書の中で、意味的に関連するグループを表す。セクションを表現するより意味的に具体的な要素がない場合に使用し、必ずではないが、通常はセクションには見出しがある。
【article（アーティクル）要素】
一つの自己完結型のコンテンツ（例えば、フォーラムの投稿、雑誌や新聞の記事、ブログの記事）を表す。必ずではないが、通常はアーティクルには見出しがある。
【aside（アサイド）要素】
文書のメインコンテンツと間接的な関係しか持っていない部分（例えば、バナー広告、関連記事、コラム）を表す。
【nav（ナビ）要素】
文書内の主要なナビゲーションを表す。

そして、「セクショニング・コンテンツ」は文書の構造を指定すると同時に「アウトライン」を明確に形作るために使用される。

4-3. アウトラインとは？

アウトライン（アウトライン・アルゴリズム）とは、文書内の構造を本の目次のように階層化したもの。

★HTML5 Outlinerで「トライデントコンピュータ専門学校」のページのアウトラインを見てみよう。

アウトラインは2種類の仕組みによって形作られる。

4-4. 暗黙的アウトライン（見出し要素）

文書内に見出し要素が出てくる度に、自動的にアウトラインが生成される。

【具体例その1】
見出しレベルが同等以下のものが出現すると「さらに細かい内容に分けられた」とみなされ、インデントのついたアウトラインが生成される。 
<h1>見出し1</h1>
<h2>見出し2</h2>
<h3>見出し3</h3>
<h4>見出し4</h4>
<h5>見出し5</h5>
<h6>見出し6</h6>
【具体例その2】
見出しレベルが同等以上のものが出現すると「次の内容に移行した」とみなされ、同列のアウトラインが生成される。 
<h1>見出し1</h1>
<h2>見出し2</h2>
<h3>見出し3</h3>
<h2>見出し2</h2>
<h3>見出し3</h3>
<h4>見出し4</h4>

★現状のHTMLのアウトラインを見てみよう。

※「h1」で囲ったロゴのラベルが表示されないが、原因は後で説明するので、いったん無視してください。

デザインが複雑になり、解釈できる媒体が増えるにつれて「暗黙的アウトライン」だけでは綺麗なアウトラインが作れなくなってきた。

4-5. 明示的アウトライン（セクショニング・コンテンツ）

セクショニング・コンテンツに属する要素が出てくる度に、自動的にアウトラインが生成される。

【具体例その1】
要素の外側に別のセクショニング・コンテンツが出現すると「次の内容に移行した」とみなされ、同列のアウトラインが生成される。 
<section>
</section>
<article>
</article>
<aside>
</aside>
<nav>
</nav>
【見出しの設定】
セクショニング・コンテンツ内の一番最初の見出しがそのアウトラインのラベルとなる。 
<section>
  <h1>見出し1</h1>
</section>
<article>
　<h2>見出し2</h2>
</article>
<aside>
</aside>
<nav>
</nav>

※一番最初の謎のアウトラインは後で説明するので、いったん無視してください。

【具体例その2】
セクショニング・コンテンツ内に、さらにセクショニング・コンテンツが出現すると「さらに細かい内容に分けられた」とみなされ、インデントのついたアウトラインが生成される。 
<section>
  <h1>見出し1</h1>
  <section>
  　<h2>見出し2</h2>
  </section>
  <aside>
  </aside>
</section>
<article>
　<h2>見出し2</h2>
</article>
<aside>
</aside>
<nav>
</nav>

デザイン的に見出しがない場合でも、「明示的アウトライン」を意識してマークアップをすれば、人間にも機械（PC）にもわかりやすいアウトラインを作ることができる！

ただし、「適当に見出しをつけても良い」ということではなく、セクショニング・コンテンツがなくても綺麗なアウトラインになるように見出しをつけること。

4-6. セクショニング・ルート

セクショニングルートは、それ自体が独自のアウトラインを形成する要素で、文書全体のアウトラインには影響を与えない。

【body要素】
一番最初にアウトラインを生成し、最初に出てきた「子要素の見出し」をラベルとする。 
<body>
  <section>
    <h1>見出し1</h1>
    <section>
    　<h2>見出し2</h2>
    </section>
    <aside>
    </aside>
  </section>
  <article>
  　<h2>見出し2</h2>
  </article>
  <aside>
  </aside>
  <nav>
  </nav>
  <h1>子要素の見出し</h1>
</body>
【blockquote要素】
「引用」を表す「blockquote要素」などは、見出しがあってもアウトラインには表示されない。 
<body>
  <section>
    <h1>見出し1</h1>
    <section>
    　<h2>見出し2</h2>
    </section>
    <aside>
      <blockquote>
        <h2>引用の見出し</h2>
      </blockquote>
    </aside>
  </section>
  <article>
  　<h2>見出し2</h2>
  </article>
  <aside>
  </aside>
  <nav>
  </nav>
  <h1>子要素の見出し</h1>
</body>

4-7. 関連項目「画像の代替テキスト」

代替テキストはWebサイト内に含まれている画像や動画が何らかの原因で表示されなかったときや、視覚障害者などが音声ブラウザを使用するときに替わりに表示するテキストを指す。

【alt（オルト）属性】
<img src="images/img_logo.png" alt="EDUCATION">

<!--
  特に代替テキストが必要ない画像でも、alt属性を追加し、値を空にしておくこと！
  ※画像にalt属性が無いと文法的にエラーとなります。
  ※「alt属性」の値を空にしておくと、音声ブラウザなどで読み飛ばしてくれるため、アクセシビリティが向上します。
-->
<img src="images/pic_trending01.png" alt="">

5-1. 子孫セレクタ

今回のデザインをよく見ると、h2となる見出しのデザインが微妙に異なっている。全てにclass名をつけても良いが、より大きな親要素にclass名をつけ「子孫セレクタ」と呼ばれる方法で指定したほうが、後々効率が良くなることが多い。 

/* 要素セレクタには、共通したプロパティを指定 */
h2{
  color: #202124;
  font-size: 24px;
  line-height: 1.2em;
  margin: 0 0 30px;
}

/* 子孫セレクタで、特定の親要素の中にある要素のみを指定 */
.worldnews h2{
  padding: 0 0 30px;
  color: #fff;
  border-bottom: 2px solid;
}

.morenews h2{
  padding: 0 0 30px;
  border-bottom: 2px solid;
}
