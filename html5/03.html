3. 要素に名前をつける（id属性とclass属性）

3-1. id属性

  • 「固有名を割り当てる」
  • 同じid名は、1ページ中に1度しか使えない 
    <!-- HTMLの書き方 -->
<h1 id="hoge">

    /* CSSの書き方 */
h1#hoge {
  color: #f00;
}

/* 要素名は省略できる */
#hoge {
  color: #f00;
}

3-2. class属性

  • 「種別名を割り当てる」
  • 同じclass名を、1ページ中に何度でも使える 
    <!-- HTMLの書き方 -->
<h2 class="fuga">
<p class="fuga">

    /* CSSの書き方 */
h2.fuga {
  color: #0f0;
}

p.fuga {
  color: #0f0;
}

/* 要素名は省略できる（class名が付いたすべての要素） */
.fuga {
  color: #0f0;
}

※プログラムで特に意味のない名前をつけたい時、日本では「hoge」「fuga」「piyo」、海外では「foo」「bar」がよく使われる。

3-3. 関連事項

【「id」のほうが「class」よりも優先度が高い】
<p id="hoge" class="fuga">
p{
  color: #f00;
}

#hoge {
  color: #0f0; /* 緑が適応される */
}

.fuga {
  color: #00f;
}
【同じ優先度の場合、cssファイル内で後に書かれたものが適応される】
<!-- id名やclass名は「半角スペース」を開けることで複数指定できる -->
<p id="hoge" class="fuga piyo">
p{
  color: #f00;
}

.fuga {
  color: #0f0;
}

.piyo {
  color: #00f; /* 青が適応される */
}
【名前の付け方】
  • 「aaa」などの意味のない文字列は使用しない
  • 「aka」「dai」「syou」「midashi」などの、日本語をローマ字にしただけの名前はかっこ悪いので使用しない
  • 「red」「big」など、デザインが変わった際にややこしくなるような名前は使用しない
  • CSSのクラス名を決めるときに使うリストをつくりました
  • codic

※最近では、デザインをパーツに分け使い回す傾向にあるため、すべての名前に「class属性」を使うのがほとんど。

4. ボックスモデル

【ボックスモデルの概念】
  • 文書内のすべての要素は、ボックスと呼ばれる四角形の領域を生成する。
  • ボックスはテキストや画像などの内容領域（Content Area）を持っており、内容領域の横幅をwidth(ウィドゥス)、高さをheight(ハイト)のプロパティで指定する。
  • その周辺には、padding(パディング)、border(ボーダー)、margin(マージン)の順に周辺領域を持つ。

4-1. Chromeデベロッパーツールで可視化する

デベロッパーツールはGoogleChromeに標準で搭載されているデバッグツールで、WebサイトのHTML構成や表示スピード、通信状況などを確認できる、超絶便利でWebサイト制作に欠かせないツール。

今回は、デベロッパーツールを使って「ボックスモデル」を説明しながらサンプルサイトを再現していきます。

【起動方法】
  • ページ上の適当な場所で右クリックし「検証」
  • ショートカット「F12」もしくは「Ctrl + Shift + I」（Macの場合は「Option + Command + I」）
【要素の選択】
  • 「Elements」タブが選択されていることを確認し、調べたい要素をクリック（▶︎をクリックすると要素内を展開できる）
  • 下図のアイコンをクリックし、Webページ上で調べたい箇所をマウスオーバーする
  • Webページは、テキストや画像などのコンテンツが何も無い部分も、必ずどれかの要素のボックスである。
  • Webページは、このボックスモデルのプロパティの値をそれぞれ指定することによって、大まかなレイアウトを形作っていく。
  • 可視化をしなくても、どの要素の「width」「margin」「padding」なのかが分かるようになると、マークアップが速く綺麗に書けるようになる。

※画像だけは例外的に何にも属さない余白がある。難しい概念なので詳細は後日。

6. width（ウィドゥス）

要素の横幅を設定する。

6-1. 単位

p{
  width: 100px;
  width: 10em; /* フォントサイズの何倍か（めったに使わない） */
  width: 100%; /* 親要素の横幅の何%か */
  width: auto; /* 親要素の横幅に対して残りの部分を自動的に埋める（初期値） */
}

【初期値とは】
プロパティを指定しなかった際にとる値のこと。

※最近のWebサイトはウィンドウサイズによってデザインや内容が変わる「レスポンシブデザイン」のサイトがほとんどなので、「%」で指定することが多い。

7. 要素の親子関係

HTMLの要素は、ある要素がある要素を含み、さらにその要素が別の要素を含む、というように階層的な構造であらわされ、この関係は一般的に親要素、子要素、孫要素などと、親子関係にたとえられて表現される。

  • 「li要素」の親要素は「ul要素」（「li要素」は「ul要素」の子要素であり、「body要素」の孫要素でもある）
  • 「ul要素」の親要素は「body要素」
  • 「body要素」の親要素は「html要素」
  • 「html要素」の親要素は無い、強いて言えば「ブラウザそのもの」

8. height（ハイト）

要素の高さを設定する。

8-1. 単位

p{
  height: 100px;
  height: 10em; /* フォントサイズの何倍か（めったに使わない） */
  height: 100%; /* 親要素の高さの何%か */
  height: auto; /* 親要素の高さに対して残りの部分を自動的に埋める（初期値） */
}

※理解せぬまま「height」を指定すると、テキストなどの量が多くなった際にはみ出してしまうため、今の所は指定しない。（サンプルサイトも使用していない）

9. margin（マージン）

ボックスの一番外側の余白の領域。他の要素との間の余白を指定する。

9-1. 指定方法

p{
  /* 個別指定 */
  margin-left: 10px;
  margin-right: 20px;
  margin-top: 30px;
  margin-bottom: 40px;

  /* 上下左右すべて */
  margin: 10px;

  /* 上下 左右 */
  margin: 10px 20px;

  /* 上 左右 下 */
  margin: 10px 20px 30px;

  /* 上 右 下 左（時計回り）*/
  margin: 10px 20px 30px 40px;
}

9-2. 単位

p{
  margin-top: 10px;
  margin-bottom: 2em; /* フォントサイズの何倍か */
  margin-left: 20%; /* 親要素の横幅の何%か（上下の余白も同様） */
  margin-right: auto; /* 親要素の横幅に対して残りの部分を自動的に埋める */
}

/*
初期値は「margin: 0」
-値が「0」の時は単位を省略できる
*/

※上下の余白は「px」か「em」、左右の余白は「%」か「auto」を使うことが多い。

9-3. マージンの相殺

マージンの相殺は、垂直マージン（上下のマージン）を指定した2つの要素が並んだ時に起こる。マージンの相殺が起こると、2つのマージンのうち大きい方（等しい場合はいずれか）が1つのマージンとしてみなされる。

以下3つのケースをディベロパーツールで実際に確認してみる。

【ケース1: 親要素と最初の子要素の上マージン】
サンプルサイト（CSSなし）の「body要素」と「h1要素」
【ケース2: 親要素と最後の子要素の下マージン】
サンプルサイト（CSSなし）の「body要素」と最後の「p要素」
【ケース3: 上下に重なり合う2つの要素のマージン】
サンプルサイト（CSSなし）の「h1要素」とその下の「p要素」など

※相殺が起こる理由は「そのほうが簡潔に書けるから」。

10. padding（パディング）

内容領域とボーダーの間にある余白の領域。要素の内側の余白を指定する。

10-1. 指定方法

p{
  /* 個別指定 */
  padding-left: 10px;
  padding-right: 20px;
  padding-top: 30px;
  padding-bottom: 40px;

  /* 上下左右すべて */
  padding: 10px;

  /* 上下 左右 */
  padding: 10px 20px;

  /* 上 左右 下 */
  padding: 10px 20px 30px;

  /* 上 右 下 左（時計回り）*/
  padding: 10px 20px 30px 40px;
}

10-2. 単位

p{
  padding-top: 10px;
  padding-bottom: 2em; /* フォントサイズの何倍か */
  padding-left: 20%; /* 親要素の横幅の何%か（上下の余白も同様） */
}

/*
初期値は「padding: 0」
「padding: auto」は存在しない
*/

※上下の余白は「px」か「em」、左右の余白は「px」か「%」を使うことが多い。

【paddingとmarginの違い】
  • 内容領域とボーダーの間の余白は「padding」、要素と要素の間の余白は「margin」
  • 「padding」には背景色が反映される
  • 「padding」には「auto」がない

11. border（ボーダー）

padding の外側にある領域、いわゆる枠線。

11-1. 指定方法

p{
  border-width: 4px; /* 線の太さ */
  border-style: solid; /* 線の種類 */
  border-color: #ccc; /* 線の色 */

  /* まとめて書く */
  border: 4px solid #ccc;

  /* 上下左右それぞれに指定する */
  border-left: 4px solid #ccc;
  border-right: 4px dashed #ccc;
  border-top: 4px dotted #ccc;
  border-bottom: 4px double #ccc;

  /* より詳しく書く */
  border-left-width: 4px;
  border-left-style: solid;
  border-left-color: #ccc;
}

11-2. 初期値と線の種類

  • border -MDN-
  • border-style -MDN-
【currentcolor】
文字と同じ色
